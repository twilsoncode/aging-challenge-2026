# Models Directory

This directory contains the saved serialized models and feature selectors generated by the machine learning pipeline `paper_script_FINAL.py`. Due to the extensive cross-dataset and cell-type-specific training, the models are organized systematically within the `saved_final_ensembles/` subdirectory.

---

## 唐 The Model Naming Scheme

Because the pipeline generates **54 distinct ensemble models**, understanding the folder and file naming convention is critical. Every folder and base file is constructed using a "Lego-block" naming scheme.

### The Standard Format
Every ensemble gets its own folder named using the following structure:
**`Ensemble`** \_ **`[Dataset]`** \_ **`[Modalities]`** \_ **`[Cell_Type]`**

### ｧｩ The Variables Explained

1. **`[Dataset]`** - The data the model was trained on:
   * `OneK1K`: Trained exclusively on the OneK1K cohort.
   * `AIDA`: Trained exclusively on the AIDA cohort.
   * `Both`: Trained on a combined dataset of both OneK1K and AIDA.

2. **`[Modalities]`** - The types of data features included in the training:
   * `All_Mods`: Uses Pseudobulk (PB), Geneformer embeddings (GF), and scGPT embeddings.
   * `PB_GF`: Uses only Pseudobulk and Geneformer embeddings.
   * `PB_scGPT`: Uses only Pseudobulk and scGPT embeddings.

3. **`[Cell_Type]`** - The biological subset the model is restricted to:
   * `All`: Trained on the global (unfiltered) pseudobulk data.
   * `B_cells`, `CD4_T`, `CD8_T`, `monocytes`, `NK`: Trained strictly on data filtered for that specific cell type.

### 庁 Examples
* **`Ensemble_AIDA_PB_GF_CD4_T`**
  * *Translation:* A model trained on **AIDA** data, using **Pseudobulk and Geneformer** features, looking specifically at **CD4+ T cells**.
* **`Ensemble_Both_All_Mods_All`**
  * *Translation:* A model trained on the **combined** datasets, using **all three modalities**, looking at the **global/overall** cells.

---

## 刀 Inside Each Ensemble Folder

Inside each of the 54 folders, the files inherit the folder's prefix, followed by their specific function:

* **The Meta-Learner:** Ends in `_meta_ridge.joblib`
  * *Example:* `Ensemble_AIDA_PB_GF_CD4_T_meta_ridge.joblib` (The final Ridge regression model combining the base learners).
* **The Base Learners:** Ends in `_base_[Modality]_[Algorithm].joblib`
  * *Example:* `Ensemble_AIDA_PB_GF_CD4_T_base_PB_XGB.joblib` (The underlying XGBoost model trained on Pseudobulk data).
* **The Local Feature Selector:** Ends in `_pb_selector.joblib`
  * *Example:* `Ensemble_AIDA_PB_GF_CD4_T_pb_selector.joblib` (The specific variance threshold and K-Best pipeline fitted to AIDA CD4+ T cell pseudobulk data).

## Global Files

* `master_pb_selector.joblib`: The global feature selection pipeline fitted on the entire (overall) OneK1K training dataset during Step 1.

## How to Load Models

You can easily load any of these components using `joblib` in Python:

```python
import joblib

# Load a meta-model
meta_model = joblib.load('saved_final_ensembles/Ensemble_OneK1K_All_Mods_All/Ensemble_OneK1K_All_Mods_All_meta_ridge.joblib')

# Load a specific base learner
xgb_base = joblib.load('saved_final_ensembles/Ensemble_OneK1K_All_Mods_All/Ensemble_OneK1K_All_Mods_All_base_PB_XGB.joblib')
