# Results Directory

This directory contains the analytical outputs, performance metrics, and feature evaluations generated by the machine learning pipeline. All outputs are saved as standard `.csv` files for easy downstream analysis and visualization.

---

## 📊 Understanding the Output Identifiers

To make it easy to map the rows in these CSV files back to the saved models, the results files utilize the same naming logic found in the `models/` directory.

When looking at the output files, pay attention to these columns:
* **`Ensemble` / `Model`**: This uses the `Dataset_Modality_CellType` naming scheme (e.g., `Ensemble_Both_PB_scGPT_monocytes`). 
* **`Base_Model`**: This indicates the specific algorithm and data modality driving a prediction (e.g., `PB_XGB` for Pseudobulk XGBoost, or `scGPT_MLP` for the scGPT Neural Network).
* **`Cell_Type`**: Indicates the cellular subset being evaluated (e.g., `All`, `CD8_T`).

---

## Generated Files

The pipeline produces the following four primary result files:

### 1. `onek1k_base_learner_metrics.csv`
Contains the raw predictive performance of every individual base learner (XGB, LGBM, RF, ElasticNet, MLPs, KNNs) evaluated solely on the OneK1K test set.
* **What it tells us:** How well each isolated algorithm and modality performs on its own before being stacked.
* **Key Columns:** `Model`, `Test_Set`, `MAE`, `RMSE`, `R2`, `Pearson`, `Spearman`, `Cell_Type`.

### 2. `onek1k_base_learner_feature_importances.csv`
Contains the ranked feature importances for all base learners trained on the OneK1K dataset. 
* **Pseudobulk Models:** Extracts native `.feature_importances_` or `.coef_` (showing which specific genes drove the predictions).
* **Embedding Models (Geneformer/scGPT):** Calculates permutation importance (showing which embedding dimensions were most critical).
* **Key Columns:** `Feature`, `Importance`, `Model`, `Cell_Type`, `Rank`.

### 3. `ensemble_reliance_weights.csv`
A detailed breakdown of how the stacked ensembles weight their respective base learners. This maps the coefficients of the Ridge meta-learner for every single cross-dataset and cell-type permutation (all 54 ensembles).
* **What it tells us:** Which base learners the meta-model "trusted" the most for a given cell type and dataset.
* **Key Columns:** `Ensemble` (The specific combination, e.g., `OneK1K_PB_GF_CD8_T`), `Cell_Type`, `Base_Model` (e.g., `GF_MLP`), `Reliance_Weight`.

### 4. `final_ensemble_evaluation_report.csv`
The ultimate performance report for the stacked ensembles. This evaluates the cross-dataset generalizations across all combinations of modalities and cell types.
* **What it tells us:** How robust the ensembles are. For example, it tracks how an ensemble trained purely on OneK1K `CD4_T` cells performs when tested against unseen AIDA `CD4_T` data.
* **Key Columns:** `Model` (The specific ensemble name), `Test_Set` (Where the test data came from), `MAE`, `RMSE`, `R2`, `Pearson`, `Spearman`.
